# 物流核算与智能寻仓工具

这是一个基于 Vite + React 的物流核算工具，主要用于尾程报价解析、SKU/国家维度的渠道寻优、产品箱规核算、库存/库龄优先发货，以及订单批量优化导出。

## 主要功能

- 导入尾程报价表，自动识别 SKU、国家、仓库、渠道和运费列。
- 按 SKU + 国家查询最优仓库与最低运费渠道。
- 支持导入优先发货/库存表，按 T2、库龄、可用库存参与推荐。
- 支持导入产品箱规表，补齐长宽高、重量和装箱数量。
- 内置 DHL、GLS、DPD 尺寸、重量、周长和体积规则校验。
- 已同步 `2026德朗司海外仓邮费规则DE-5.15更新（应用版）` 中的德国境内/境外核心发货限制与附加费。
- 支持订单工作簿批量处理，输出换仓建议和最低运费结果。
- 本地缓存已导入数据，刷新页面后可继续使用。

## 数据准备

### 尾程报价表

表格中至少需要包含：

- SKU 或商品编码
- 国家或目的国
- 仓库名称
- 渠道名称
- 运费/报价数据

系统会自动识别多级表头，并跳过明显不是报价渠道的辅助列。

### 优先与库存表

建议包含：

- SKU
- 仓库
- 可用库存
- T2 或库龄相关信息

系统会同时识别 `EB-`、`AM-` 和无前缀 SKU，减少同款 SKU 因前缀不同而无法匹配的问题。

### 产品箱规表

建议包含：

- 通用 SKU / 亚马逊 SKU / eBay SKU
- 一箱件数
- 长 cm
- 宽 cm
- 高 cm
- 总发毛重 kg

页面内提供产品箱规模板下载。

## 已优化点

- 统一了费用、库存、尺寸和重量的数值解析。
- 支持 `12.34`、`12,34`、`1,234.56`、`1.234,56`、`EUR 4,30` 等常见格式。
- 避免欧式小数被错误算成整数或千位数，提升运费核算准确性。
- 更新 DPD 附加费：超尺寸/超体积 `33元`，超重 `282.1元`，超尺寸 II `279.2元`，超限 `488.9元`。
- 新增德国发境外 DHL 自动拦截：最长边 `>=150cm`、周长 `>300cm` 或重量 `>30kg` 时排除。
- 将清理脚本改为跨平台写法，Windows 下也能正常执行。
